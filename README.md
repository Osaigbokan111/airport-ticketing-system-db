Airport Ticketing System Database
Project Overview
This project is a complete relational database design for an Airport Ticketing System.
It covers flight scheduling, passenger reservations, payments, ticket issuance, and staff access control.
Technologies Used
Microsoft SQL Server
T-SQL
Relational Database Design
Stored Procedures
Triggers
Views
Role-Based Security
Features
Normalized database schema
Strong data integrity using constraints
Automated workflows using triggers
Revenue reporting with views and stored procedures
Concurrency control to prevent double booking
Role-based access (Staff & Supervisor)
Folder Structure
`database/` → schema, tables, triggers, views
 `data/` → seed/sample data
 `queries/` → test and reporting queries
`diagrams/` → ER diagram
How to Run
Create database using `01_create_database.sql`
Run table scripts in order
Load seed data
Execute procedures and queries
